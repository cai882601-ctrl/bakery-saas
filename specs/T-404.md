# T-404: Supabase Auth ‚Äî Signup/Login/Logout + Route Protection

## Overview
Add user authentication to BakeBoard using Supabase Auth. Currently the dashboard is completely open with no auth. Users need to be able to sign up, log in, and log out. Dashboard routes must be protected.

## Requirements

### Functional
1. **Signup page** (`/auth/signup`) ‚Äî email + password registration via Supabase Auth
2. **Login page** (`/auth/login`) ‚Äî email + password login
3. **Logout** ‚Äî button in header/sidebar that calls `supabase.auth.signOut()`
4. **Route protection** ‚Äî all `/dashboard/*` routes redirect to `/auth/login` if not authenticated
5. **Landing page** ‚Äî "Get started" and "Sign in" buttons link to signup/login pages
6. **Auth callback** ‚Äî handle Supabase auth redirect (`/auth/callback`)
7. **Post-login redirect** ‚Äî after login/signup, redirect to `/dashboard`

### Technical
- Use `@supabase/ssr` for server-side auth (Next.js App Router compatible)
- Create Supabase client utilities: `createServerClient` and `createBrowserClient`
- Use Next.js middleware for route protection (check session, redirect if needed)
- Email confirmation can be disabled for MVP (Supabase dashboard setting)

### UI
- Auth pages should match existing design system (shadcn/ui + Tailwind)
- Simple, clean forms ‚Äî email + password + submit button
- Error messages for invalid credentials, existing accounts, etc.
- "Already have an account? Sign in" / "Don't have an account? Sign up" links

## Acceptance Criteria
1. New user can sign up at `/auth/signup` with email + password
2. Existing user can log in at `/auth/login`
3. Unauthenticated user visiting `/dashboard` is redirected to `/auth/login`
4. Authenticated user can access all dashboard pages
5. User can log out from the dashboard
6. Landing page buttons point to auth pages (not directly to `/dashboard`)
7. `npm run build` passes with no errors
8. Supabase `auth.users` table shows new registrations

- `src/components/header.tsx` or `sidebar.tsx` (update ‚Äî add logout button)

## Constraints
- Do NOT change database schema or tRPC routes
- Do NOT add social auth (Google/GitHub) ‚Äî email+password only for MVP
- Keep it minimal ‚Äî no password reset, no email verification for now
