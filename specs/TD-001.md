# TD-001: Dashboard loading/error state handling

## Goal
Dashboard must distinguish between loading, error, and genuine empty states so users see appropriate feedback instead of a misleading onboarding checklist.

## User Stories
- As a user with existing data, I want to see a loading indicator while data fetches so I don't think my account is empty.
- As a user experiencing a network error, I want to see an error message so I know something went wrong.

## Acceptance Criteria
- [ ] AC-1: All 4 tRPC queries in dashboard/page.tsx destructure `isLoading` and `isError`
- [ ] AC-2: When any query is loading, show a loading skeleton or spinner instead of "0" stats
- [ ] AC-3: When any query errors, show an error message with retry option instead of onboarding checklist
- [ ] AC-4: Onboarding checklist only appears when all queries have successfully returned AND all counts are genuinely 0
- [ ] AC-5: Also fix TD-002: Active Products count uses a dedicated count query or removes limit:1 to get accurate count

## Technical Constraints
- Stack: Next.js 15 + tRPC + Supabase + shadcn/ui + Tailwind
- Routers: src/server/routers/
- Schema: src/db/schema.ts
- Do NOT modify: .env files
- Dashboard page: src/app/(dashboard)/dashboard/page.tsx
- Products router: src/server/routers/products.ts (may need count endpoint or remove limit)

## Out of Scope
- TD-003 (Record<string, unknown> type safety) — separate task
- TD-004 (Settings page) — separate task
- Toast notifications / loading skeletons for other pages
